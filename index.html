Security Mindset

The security mindset is a perspective that emphasises the susceptibility of security systems to failure and breach.





CIA

  • Confidentiality
    Protecting data from those who should not have access to it.


  • Integrity
    Data is maintained in its orginal form, without unwanted change.


  • Availability
    Users have any-time, any-place access to data.
    E.g. of breach of availability: denial of service (DoS) attack.

Protection of Systems - AAA

Protection of systems is aimed at ensuring accountability and is premised on the three A's:

  • Authentication
    Passwords, biometrics and tokens

  • Authorisation
    Access control

  • Audit (Accounting)
    Intrusion detection & forensic analysis






Threats, Vulnerabilities and Risk

Security attacks are categorised in terms of Threats, Vulnerabilities, Risk and Impact.

Threats Vulnerabilities Risk Impact
Events or entities that have potential to harm business or personal assets. Weaknesses in the security system that can be exploited by malicious entities. The likelihood that a malicious event will occur. Impact of any event depends on the value of the assets affected; and the extent to which these assets have been affected.
E.g. Insider threat






Controls

The types of security measures that can be implemented are categorised in terms of the Physical, Logical and Administrative realms.

Physical Logical (technical) Administrative
Security Guards, gates, CCTV cameras, locks Passwords, firewalls, encryption, etc. Policies, mandatory procedures, laws, etc.
Physical barriers to access of data Prevents unauthorised access to data and manipulation thereof Creates framework for appropriate conduct of users






Security Principles

  • Economy of mechanism
    Design of security measures should be simple and implementation of these measures should not unduly compromise usability.

  • Fail-safe defaults
    Default configuration should have a conservative protection scheme.

  • Open design
    Security architecture and design should be publicly accessible.

  • Least privilege
    Programs and/or users should have minimum (administrative) rights to do their job.







Malware

Malware (malicious software) can be categorised based on propagation and concealment. Malware has manfifold payloads - ranging from annoyance to crime.

Propagation Concealment
Virus: human-assisted propagation (embedded in email attachment) Rootkit: modifies operating system to hide its existence
Worm: automatic propagation without human assistance Trojan: provides desirable functionality but obscures malicious operation






Examples of Malware

Insider Attacks Backdoors
An insider attack is a breach of security that is conducted or enabled by someone who is part of the organisation that controls or builds the asset that should be protected. A backdoor (or trapdoor ) is a hidden feature or command in a program that enables a user to perform actions which they are not normally authorised or permitted to do - actions for which ordinary users do not have privilege.
In the case of malware, an insider attack refers to a security hole that is created in a software system by one of the software programmers. When used in the permitted way, this program behaves as expected with only the mandated privileges.
But if the hidden feature is activated, the program executed unexpected (such as performing a privilege escalation)- usually in contravention of security policies.
Benign example: Easter Eggs in DVDs and software
